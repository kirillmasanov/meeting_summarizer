# 🎥 Meeting Summarizer

Веб-приложение для автоматической обработки видеозаписей встреч с извлечением аудио, асинхронной транскрибацией через Yandex SpeechKit API v3 и суммаризацией через AI Studio Model Gallery на основе настраиваемого системного промпта.

## 🌟 Возможности

- **Загрузка видео**: Drag-and-drop интерфейс для загрузки видеофайлов (MP4, WebM, до 1GB и длительностью до 4 часов)
- **Извлечение аудио**: Автоматическое извлечение аудиодорожки из видео с помощью ffmpeg
- **Асинхронная транскрибация**: Преобразование речи в текст через Yandex SpeechKit API v3
- **Суммаризация**: Создание структурированного резюме встречи через AI Studio Model Gallery
- **Настраиваемый промпт**: Возможность задать собственные инструкции для суммаризации
- **Копирование результатов**: Быстрое копирование резюме в буфер обмена
- **Многопользовательская работа**: Каждый пользователь работает в своей изолированной сессии
- **Гибкая настройка API ключей**: Возможность использовать как env переменные, так и ввод через веб-интерфейс
- **Валидация ключей**: Автоматическая проверка корректности API ключа перед использованием

## 📋 Требования

- **Python 3.8+**

```bash
git clone https://github.com/kirillmasanov/meeting-summarizer
```

**Примечание:** При использовании `uv` не требуется вручную активировать виртуальное окружение. Команда `uv run` автоматически использует созданное окружение.

### Настройка Yandex Cloud

#### 1. Получение Folder ID

Folder ID можно получить с помощью yc cli:
```bash
yc config get folder-id
```

Или в веб-консоли:
1. Перейдите в [консоль Yandex Cloud](https://console.cloud.yandex.ru/)
2. Выберите ваш каталог
3. ID каталога отображается в URL или в информации о каталоге

#### 2. Создание сервисного аккаунта

**Через веб-консоль:**
1. Перейдите в раздел "Сервисные аккаунты"
2. Нажмите "Создать сервисный аккаунт"
3. Укажите имя (например, `meeting-summarizer-sa`)
4. Добавьте описание (опционально)

**Через CLI:**
```bash
# Создание сервисного аккаунта
yc iam service-account create \
  --name meeting-summarizer-sa \
  --description "Service account for meeting summarizer app"

# Получение ID созданного аккаунта
SA_ID=$(yc iam service-account get meeting-summarizer-sa --format json | jq -r '.id')
echo "Service Account ID: $SA_ID"
```

#### 3. Назначение ролей

Сервисному аккаунту необходимы следующие роли:
- `ai.speechkit-stt.user` - для работы с SpeechKit API (асинхронная транскрибация)
- `ai.languageModels.user` - для работы с AI Studio Model Gallery (суммаризация)

**Через веб-консоль:**
1. Откройте страницу каталога
2. Перейдите на вкладку "Права доступа"
3. Нажмите "Назначить роли"
4. Выберите сервисный аккаунт
5. Добавьте обе роли: `ai.speechkit-stt.user` и `ai.languageModels.user`

**Через CLI:**
```bash
# Получите FOLDER_ID
FOLDER_ID=$(yc config get folder-id)

# Назначьте роли
yc resource-manager folder add-access-binding $FOLDER_ID \
  --role ai.speechkit-stt.user \
  --service-account-id $SA_ID

yc resource-manager folder add-access-binding $FOLDER_ID \
  --role ai.languageModels.user \
  --service-account-id $SA_ID
```

#### 4. Создание API ключа

**Через веб-консоль:**
1. Откройте страницу сервисного аккаунта
2. Перейдите на вкладку "API-ключи"
3. Нажмите "Создать API-ключ"
4. Скопируйте ключ (он показывается только один раз!)

**Через CLI:**
```bash
# Создание API-ключа
yc iam api-key create --service-account-id $SA_ID --format json

# Сохранение ключа в переменную
API_KEY=$(yc iam api-key create --service-account-id $SA_ID --format json | jq -r '.secret')
```

#### 5. Настройка переменных окружения

##### Способ 1: Через переменные окружения

Создайте файл `.env` в корне проекта:

```env
API_KEY=your_api_key_here
FOLDER_ID=your_folder_id_here
LLM_MODEL=qwen3-235b-a22b-fp8/latest
SERVER_PORT=8000
```

**Примечание:** Параметры `LLM_MODEL` и `SERVER_PORT` опциональны. Если не указаны, будут использованы значения по умолчанию.

##### Способ 2: Через веб-интерфейс

Если переменные `API_KEY` и `FOLDER_ID` не заданы в `.env`, при первом открытии приложения появится форма для ввода ключей:

1. Откройте приложение в браузере
2. Введите `API_KEY` и `FOLDER_ID` в форму
3. Нажмите "Сохранить и продолжить"
4. Ключи будут валидированы и сохранены в вашей сессии

**Преимущества способа 2:**
- Не требуется создавать `.env` файл
- Каждый пользователь может использовать свои ключи
- Удобно для тестирования с разными аккаунтами
- Ключи хранятся только в памяти сервера (не на диске)

**Автоматическое создание .env через CLI:**
```bash
# Получение значений
FOLDER_ID=$(yc config get folder-id)
SA_ID=$(yc iam service-account get meeting-summarizer-sa --format json | jq -r '.id')
API_KEY=$(yc iam api-key create --service-account-id $SA_ID --format json | jq -r '.secret')

# Создание .env файла
cat > .env << EOF
API_KEY=$API_KEY
FOLDER_ID=$FOLDER_ID
LLM_MODEL=qwen3-235b-a22b-fp8/latest
SERVER_PORT=8000
EOF
```

## 📖 Использование

### Веб-интерфейс

#### Первый запуск

1. Откройте **http://localhost:8000** в браузере
2. **Если API ключ и ID каталога не заданы в `.env`:**
   - Появится форма для ввода
   - Введите `API_KEY` и `FOLDER_ID`
   - Нажмите "Сохранить и продолжить"
   - Ключи будут автоматически валидированы
3. **Если API ключ и ID каталога заданы в `.env`:**
   - Форма не появится, можно сразу загружать файлы

#### Обработка видео

1. Перетащите видеофайл в область загрузки или нажмите для выбора
2. (Опционально) Настройте системный промпт в разделе "Настройки промпта"
3. Нажмите "Загрузить и обработать"
4. Дождитесь завершения обработки (отображается спиннер с текущей стадией):
   - 📤 Загрузка файла
   - 🎵 Извлечение аудио
   - 🎤 Транскрибация и суммаризация
5. Просмотрите резюме встречи
6. Скопируйте результаты в буфер обмена при необходимости
7. Нажмите "Обработать новый файл" для следующей задачи

### Многопользовательская работа

Приложение поддерживает одновременную работу нескольких пользователей:

- **Изолированные сессии**: Каждый пользователь работает в своей сессии
- **Собственные API ключи**: Каждый может использовать свои ключи (если не заданы в env)
- **Независимые задачи**: Задачи одного пользователя не влияют на других
- **Автоматическая очистка**: Неактивные сессии удаляются через 30 минут

**Пример использования:**
- Пользователь A открывает приложение → создается сессия A
- Пользователь B открывает приложение → создается сессия B
- Оба могут загружать и обрабатывать видео независимо
- Каждый видит только свои задачи и результаты

## ⚙️ Конфигурация

### Ограничения

- **Максимальный размер файла**: 1 GB
- **Максимальная длительность видео**: 4 часа (240 минут)
- **Поддерживаемые форматы**: MP4, WebM (Zoom, Телемост)
- **Хранение данных**: Файлы и статусы удаляются сразу после получения результата
- **Аудио формат**: Автоматическое преобразование в MP3

### Настройка переменных окружения

Приложение поддерживает следующие переменные окружения в файле `.env`:

- `API_KEY` (опционально*) - API ключ для Yandex Cloud
- `FOLDER_ID` (опционально*) - ID каталога в Yandex Cloud
- `LLM_MODEL` (опционально, по умолчанию: `qwen3-235b-a22b-fp8/latest`) - модель для суммаризации
- `SERVER_PORT` (опционально, по умолчанию: `8000`) - порт для запуска сервера

**\*Примечание:** Если `API_KEY` и `FOLDER_ID` не заданы в `.env`, они могут быть введены через веб-интерфейс при первом использовании.

### Архитектура сессий

Приложение использует session-based архитектуру для поддержки многопользовательской работы:

```
┌─────────────────┐
│  Пользователь 1 │──► Session ID: abc123
│  (API Key A)    │    └─► Task 1, Task 2
└─────────────────┘

┌─────────────────┐
│  Пользователь 2 │──► Session ID: def456
│  (API Key B)    │    └─► Task 3
└─────────────────┘

┌─────────────────┐
│  Пользователь 3 │──► Session ID: ghi789
│  (Default Keys) │    └─► Task 4, Task 5
└─────────────────┘
```

**Особенности:**
- Session ID хранится в httpOnly cookie
- TTL сессии: 30 минут с момента последней активности
- Автоматическая очистка неактивных сессий
- Задачи привязаны к сессиям и изолированы друг от друга

### Настройка системного промпта

Системные промпты задают формат резюме и загружаются из папки `prompts/` при запуске приложения. По умолчанию доступны: «Встреча с клиентом» (`prompts/meeting.txt`) и «Обучающий вебинар» (`prompts/webinar.txt`). Выбор типа промпта — в веб-интерфейсе.

Чтобы изменить текст промпта, отредактируйте соответствующий файл в `prompts/` и перезапустите приложение. 

## 🐳 Docker

### Особенности Docker-версии

- **Автоматическая установка ffmpeg** - не требуется устанавливать вручную
- **Изолированное окружение** - все зависимости внутри контейнера
- **Простое развертывание** - один файл `.env` (опционально) и команда `docker compose up`
- **Персистентность данных** - директории `uploads/` и `temp/` монтируются как volumes

### Управление контейнером

```bash
# Пересборка образа после изменений
docker compose build

# Запуск в фоновом режиме
docker compose up -d

# Просмотр логов
docker compose logs -f meeting-summarizer

# Остановка без удаления контейнера
docker compose stop

# Остановка и удаление контейнера
docker compose down

# Полная очистка (включая volumes)
docker compose down -v
```

### Настройка портов

По умолчанию приложение доступно на порту 8000. Чтобы изменить порт, укажите `SERVER_PORT` в файле `.env`:

```env
SERVER_PORT=3000
```

Затем пересоздайте контейнер:
```bash
docker compose up -d --force-recreate
```

## 🔄 Обновление приложения на виртуальной машине

Подключитесь к вашей ВМ по SSH и выполните следующие команды:

```bash
# 1. Перейдите в директорию проекта
cd /path/to/meeting-summarizer

# 2. Остановите текущий контейнер
```

### Проверка обновления

После обновления убедитесь, что все работает корректно:

```bash
# Проверьте последний коммит
git log -1 --oneline

# Проверьте статус контейнера
docker compose ps

# Откройте приложение в браузере
# http://<IP-вашей-ВМ>:8000
```

### Важные моменты

- **Файл .env сохраняется**: Ваши настройки в `.env` не будут затронуты при обновлении
- **Сессии сбрасываются**: При перезапуске приложения все активные сессии будут сброшены
- **Данные сохраняются**: Файлы в `uploads/` и `temp/` (если есть) сохранятся благодаря volumes

### Откат к предыдущей версии

Если что-то пошло не так, можно откатиться к предыдущей версии:

## 🔒 Безопасность

### Рекомендации для production

1. **Используйте HTTPS**: Настройте reverse proxy (nginx/traefik) с SSL сертификатом
2. **Ограничьте доступ**: Используйте firewall или VPN для ограничения доступа
3. **Храните ключи в env**: Не вводите production ключи через веб-интерфейс
4. **Мониторинг**: Отслеживайте использование API и логи приложения
5. **Регулярные обновления**: Обновляйте зависимости для устранения уязвимостей

### Хранение данных

- **API ключи**: Хранятся в памяти (не на диске) в рамках сессии
- **Видео файлы**: Удаляются сразу после извлечения аудио
- **Аудио файлы**: Удаляются сразу после транскрибации
- **Результаты**: Удаляются после получения клиентом
- **Сессии**: Автоматически очищаются через 30 минут неактивности
